Critical OOP Fundamentals P0 0/8
Classes and Objects

Classes are blueprints for objects. Objects are instances of classes containing state (fields) and behavior (methods). The foundation of all Java programming.

class new instance
Constructors and Initialization

Special methods for object initialization. Default constructors, parameterized constructors, constructor chaining with this() and super().

constructor this() super()
Inheritance (extends)

Child classes inherit fields and methods from parent classes. Enables code reuse and establishes IS-A relationships. Single inheritance only in Java.

extends super IS-A
Polymorphism

Compile-time (method overloading) and runtime (method overriding) polymorphism. Same interface, different implementations. Core to Spring's DI.

@Override Overloading Dynamic Dispatch
Interfaces (implements)

Contracts defining what a class must do without specifying how. Multiple interface implementation allowed. Default and static methods since Java 8.

interface implements default
Abstract Classes vs Interfaces

Abstract classes can have state and partial implementation. Interfaces define pure contracts. Use abstract for shared code, interfaces for capabilities.

abstract When to Use
Encapsulation (Access Modifiers)

Control visibility with public, private, protected, package-private. Hide implementation details, expose only what's necessary. Getters/setters pattern.

public private protected
Composition over Inheritance

Favor HAS-A relationships over IS-A. More flexible, loosely coupled designs. Dependency injection relies heavily on composition.

HAS-A Delegation Flexibility
Critical Modern Java Essentials P1 0/2
Records (Java 14+)

Immutable data carriers with auto-generated constructors, getters, equals(), hashCode(), toString(). Perfect for DTOs in Spring Boot APIs.

record Immutable DTO
var Keyword and Type Inference

Local variable type inference with var. Compiler infers type from initializer. Reduces verbosity while maintaining type safety.

var Type Inference Java 10+
Backend Essential Collections Framework P2 0/6
List Interface (ArrayList, LinkedList)

Ordered collections allowing duplicates. ArrayList for O(1) random access. LinkedList for O(1) insertions/deletions. Most common collection in APIs.

ArrayList LinkedList List.of()
Map Interface (HashMap, TreeMap)

Key-value pairs. HashMap provides O(1) lookup - essential for caching. TreeMap maintains sorted order. LinkedHashMap preserves insertion order.

HashMap TreeMap O(1) lookup
Set Interface (HashSet, TreeSet)

Unique elements only. HashSet for O(1) contains check. TreeSet for sorted unique elements. Essential for deduplication.

HashSet TreeSet No Duplicates
Queue and Deque

FIFO ordering for task queues. LinkedList implements Queue. ArrayDeque for double-ended operations. Critical for message processing in backends.

Queue Deque FIFO
Iterator and For-Each

Safe iteration over collections. Iterator allows removal during iteration. Enhanced for-loop for cleaner code. Iterable interface.

Iterator for-each Iterable
Collections Utility Class

Static methods for sorting, searching, shuffling, reversing. Collections.unmodifiableList() for immutable wrappers. synchronizedList() for thread safety.

Collections.sort() unmodifiable synchronized
Backend Essential Exception Handling P3 0/4
Checked vs Unchecked Exceptions

Checked exceptions must be handled (IOException). Unchecked extend RuntimeException (NullPointerException). APIs typically throw unchecked for cleaner code.

Exception RuntimeException throws
Try-Catch-Finally Blocks

try for risky code, catch for handling, finally for cleanup. Multiple catch blocks. Catch specific exceptions before general ones.

try catch finally
Try-With-Resources

Automatic resource management for AutoCloseable resources. Ensures streams, connections are closed even if exceptions occur. Essential for DB connections.

try-with-resources AutoCloseable Java 7+
Custom Exceptions

Domain-specific exceptions (UserNotFoundException, InvalidOrderException). Extend RuntimeException for unchecked. Include meaningful messages and error codes.

extends Exception Domain Errors
Backend Essential Concurrency Basics P4 0/2
Thread and Runnable

Thread class for concurrent execution. Runnable interface for task definition. Understand thread states: NEW, RUNNABLE, BLOCKED, WAITING, TERMINATED.

Thread Runnable start()
ExecutorService and Thread Pools

Manage thread pools instead of creating threads manually. Executors.newFixedThreadPool() for bounded pools. Submit tasks, get Futures. Essential for async processing.

ExecutorService ThreadPool Future
Important Functional Programming P5 0/5
Lambda Expressions

Anonymous functions with concise syntax: (params) -> expression. Enable functional programming style. Used extensively in Spring Security DSL and Stream API.

-> Anonymous Function Java 8+
Functional Interfaces

Single abstract method interfaces. Predicate for boolean tests, Function for transformations, Consumer for actions, Supplier for providers.

Predicate Function Consumer Supplier
Method References (::)

Shorthand for lambdas calling existing methods. String::toUpperCase, System.out::println, User::new. Cleaner than equivalent lambdas.

:: Reference Shorthand
Optional Class

Container for potentially null values. Avoid NullPointerException with Optional.ofNullable(), orElse(), orElseThrow(). Common in repository return types.

Optional orElse Null Safety
Immutability Principles

Objects that cannot change after creation. final fields, no setters, defensive copies. Thread-safe by default. Records are immutable. Prefer for DTOs.

final Immutable Thread-safe
Important Streams API P6 0/5
Creating Streams

From collections: list.stream(). From values: Stream.of(). From arrays: Arrays.stream(). Infinite streams: Stream.generate(), Stream.iterate().

.stream() Stream.of() generate()
Intermediate Operations

Lazy operations returning streams. map() transforms, filter() selects, flatMap() flattens nested, sorted() orders, distinct() removes duplicates.

map() filter() flatMap() sorted()
Terminal Operations

Trigger processing and return result. collect() gathers results, reduce() aggregates, forEach() performs action, findFirst()/findAny() retrieves elements.

collect() reduce() forEach() findFirst()
Collectors Utility

Common collection strategies. toList(), toSet(), toMap() for basic collections. groupingBy() for grouping, joining() for strings, counting() for aggregates.

Collectors.toList() groupingBy() toMap()
Parallel Streams

Automatic parallelization with parallelStream(). Uses ForkJoinPool. Best for CPU-intensive tasks on large datasets. Watch for side effects and ordering.

parallelStream() ForkJoinPool Parallel
Good to Know Advanced Modern Java P7 0/4
Sealed Classes and Interfaces

Restrict which classes can extend/implement. permits clause defines allowed subtypes. Enables exhaustive pattern matching in switch.

sealed permits Java 17+
Pattern Matching for instanceof

Combine type check and cast: if (obj instanceof String s). Eliminates explicit casting. Cleaner polymorphic code.

instanceof Pattern Java 16+
Switch Expressions

Arrow syntax (case X -> value) returns values directly. yield for multi-line cases. Exhaustiveness checking. Much cleaner than switch statements.

switch -> yield Java 14+
Text Blocks

Multiline strings with """ delimiters. Preserves formatting, handles escapes. Perfect for JSON, SQL, HTML templates in code.

""" Multiline Java 15+
Good to Know I/O and Files P8 0/3
File and Path Classes (NIO.2)

Path for file system paths, Files for operations. Modern replacement for java.io.File. Paths.get(), Files.exists(), Files.createDirectory().

Path Files NIO.2
Reading/Writing Files

Files.readString(), Files.writeString() for text. Files.readAllLines() for line-by-line. Files.lines() returns Stream for lazy processing.

readString() writeString() lines()
BufferedReader and BufferedWriter

Efficient I/O with buffering. Wrap InputStreamReader/OutputStreamWriter. Use with try-with-resources for automatic closing.

BufferedReader BufferedWriter Buffering
Good to Know Generics P9 0/3
Generic Classes and Methods

Type parameters for type-safe reusable code. class Box, T getValue(). Compile-time type checking, no casts needed.

<T> Type Parameter Type Safety
Bounded Type Parameters

Restrict type parameters: >. Upper bounds with extends, both classes and interfaces. Multiple bounds possible.

extends Bounded Comparable
Wildcards (?, extends, super)

? for unknown types. ? extends T for producers (read). ? super T for consumers (write). PECS: Producer Extends, Consumer Super.

? ? extends ? super PECS
Good to Know Advanced Concurrency P10 0/2
synchronized and volatile

synchronized blocks/methods for mutual exclusion. volatile ensures visibility across threads. Understand happens-before relationships.

synchronized volatile Thread Safety
CompletableFuture

Async programming with chaining. thenApply(), thenCompose() for transformations. exceptionally() for error handling. Combines multiple futures.

CompletableFuture thenApply() Async

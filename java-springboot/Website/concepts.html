P0 Spring Core Container 0/5
Inversion of Control (IoC)

The framework manages object creation and wiring instead of the developer. The ApplicationContext is the IoC container that manages Bean lifecycles.

ApplicationContext Container
Dependency Injection (DI)

Dependencies are "injected" into objects by the container rather than created by the object itself. Enables loose coupling and easier testing.

@Autowired Loose Coupling
Constructor vs Setter vs Field Injection

Constructor injection is preferred for mandatory dependencies (allows final fields). Field injection (@Autowired on fields) is discouraged as it hides dependencies.

Constructor Injection Best Practice
Bean Scopes

Singleton (default): one instance per container. Prototype: new instance each request. Request/Session: tied to HTTP lifecycle. Singleton beans must be stateless!

Singleton Prototype Request
Component Scanning & Stereotypes

@Component, @Service, @Repository, @Controller mark classes as Spring-managed beans. Component scanning auto-detects these annotations.

@Component @Service @Repository
P1 Spring Boot Internals 0/5
@SpringBootApplication

Combines @Configuration, @EnableAutoConfiguration, and @ComponentScan. The entry point annotation for Spring Boot applications.

Entry Point Main Class
Auto-Configuration Mechanism

Spring Boot scans classpath for libraries and auto-configures beans. Uses @Conditional annotations to decide what to configure. Debug with --debug flag.

@EnableAutoConfiguration Conditionals
@Conditional Annotations

@ConditionalOnClass, @ConditionalOnMissingBean control when auto-config activates. Allows defaults to "back off" when you define custom beans.

@ConditionalOnClass @ConditionalOnMissingBean
Spring Boot Starters

Dependency descriptors that bundle related libraries. spring-boot-starter-web pulls in Spring MVC, Jackson, Tomcat, etc. Simplifies dependency management.

starter-web starter-data-jpa
Bill of Materials (BOM)

spring-boot-dependencies manages versions of transitive dependencies. Solves "JAR hell" by ensuring compatible library versions.

Version Management Parent POM
P2 RESTful Web Services 0/6
@RestController vs @Controller

@RestController includes @ResponseBody, serializing return values directly to JSON. @Controller returns view names for template engines like Thymeleaf.

@RestController @ResponseBody
Request Mapping Annotations

@GetMapping, @PostMapping, @PutMapping, @PatchMapping, @DeleteMapping for HTTP method-specific routing. Use correct semantics!

@GetMapping @PostMapping @RequestMapping
Path and Query Parameters

@PathVariable extracts from URL path (/users/{id}). @RequestParam extracts query params (?name=value). @RequestBody deserializes JSON body.

@PathVariable @RequestParam @RequestBody
JSON Serialization (Jackson)

Jackson auto-converts Java objects to/from JSON. Use @JsonIgnore, @JsonProperty for customization. Configure ObjectMapper for date formats, etc.

Jackson ObjectMapper
DTOs vs Entities

Never expose JPA entities directly! DTOs define API contracts, decouple internal models from public API. Use Records for immutable DTOs. MapStruct for mapping.

Data Transfer Objects Records
Global Exception Handling

@ControllerAdvice + @ExceptionHandler provide centralized error handling. Return standardized error responses with proper HTTP status codes.

@ControllerAdvice @ExceptionHandler
P3 Data Persistence 0/6
Spring Data JPA

Abstraction over JPA/Hibernate. Reduces boilerplate by generating repository implementations from interface definitions at runtime.

JpaRepository CrudRepository
Entity Mapping

@Entity marks JPA entities. @Id + @GeneratedValue for primary keys. @Column for field customization. @Table for table naming.

@Entity @Id @Column
Repository Pattern

Extend JpaRepository to get full CRUD for free. Spring generates implementation at runtime using proxies.

Repository Interface Proxy
Derived Query Methods

Spring parses method names to generate queries: findByEmailAndIsActiveTrue() becomes SQL automatically. Magic method naming conventions!

findBy Query Derivation
JPQL and Native Queries

@Query annotation for custom JPQL or native SQL. Use for complex joins and optimizations. Named parameters with :paramName.

@Query JPQL Native SQL
Transaction Management

@Transactional ensures ACID properties. Understand propagation levels. Beware: calling @Transactional from same class bypasses proxy!

@Transactional Propagation
P4 Configuration Management 0/5
application.properties / yml

Central configuration files in src/main/resources. YAML offers hierarchical structure. Properties for flat key-value pairs.

application.yml application.properties
@Value Annotation

Inject single property values: @Value("${app.name}"). Supports SpEL expressions and default values: @Value("${port:8080}").

@Value SpEL
@ConfigurationProperties

Type-safe configuration binding. Map groups of properties to POJOs. Better than @Value for complex configs. Enable with @EnableConfigurationProperties.

@ConfigurationProperties Type-safe
Profiles (dev/prod)

Environment-specific configs with @Profile and application-{profile}.yml. Activate with spring.profiles.active. Prevent credential hardcoding!

@Profile application-dev.yml
Environment Variables

Override properties via env vars: SPRING_DATASOURCE_PASSWORD. 12-factor app compliance. Essential for containerized deployments.

Environment Vars 12-Factor
P5 Testing 0/5
JUnit 5

Modern testing framework. @Test, @BeforeEach, @AfterEach for lifecycle. @DisplayName for readable test names. Assertions and assumptions.

@Test Assertions
Mockito

Mock dependencies for isolated unit testing. @Mock creates mocks, @InjectMocks injects them. when().thenReturn() for stubbing behavior.

@Mock @InjectMocks when/thenReturn
@SpringBootTest

Loads full application context for integration testing. Powerful but slower. Use for end-to-end testing of multiple layers working together.

@SpringBootTest Integration Test
Slice Tests

@WebMvcTest loads only web layer, mocks services. @DataJpaTest loads only persistence layer with in-memory DB. Faster than full context.

@WebMvcTest @DataJpaTest MockMvc
Testcontainers

Spin up real databases in Docker for tests. Ensures environment parity with production. Eliminates H2 vs PostgreSQL discrepancies.

Testcontainers Docker
P6 Security 0/5
Security Filter Chain

Spring Security operates as a chain of servlet filters intercepting requests. Configure via SecurityFilterChain bean. Order matters!

SecurityFilterChain Filters
Authentication vs Authorization

Authentication: "Who are you?" (login). Authorization: "What can you do?" (permissions). AuthenticationManager validates, AccessDecisionManager authorizes.

AuthN AuthZ
UserDetailsService

Interface for loading user-specific data. Implement to fetch users from database. Returns UserDetails with username, password, authorities.

UserDetailsService UserDetails
JWT Implementation

Stateless authentication with JSON Web Tokens. Client sends token in Authorization header. Custom filter validates and populates SecurityContext.

JWT Bearer Token
OAuth2 Basics

Delegated authorization ("Login with Google"). Understand flows: Authorization Code, Client Credentials. Spring as Resource Server or OAuth2 Client.

OAuth2 OpenID Connect
P7 Observability 0/4
Spring Boot Actuator

Production-ready features: /health, /metrics, /info endpoints. Monitor application state. Configure exposure and security in production.

Actuator /health /metrics
Health Endpoints

Custom health indicators for dependencies. Aggregate health status. Kubernetes liveness/readiness probes integration.

HealthIndicator Probes
Micrometer Metrics

"SLF4J for metrics" - vendor-neutral instrumentation. Ships to Prometheus, Datadog, New Relic without code changes. Counters, gauges, timers.

Micrometer Prometheus
Logging (SLF4J, Logback)

SLF4J as logging facade, Logback as implementation. Configure levels, appenders, patterns in logback-spring.xml. MDC for request tracing.

SLF4J Logback MDC
P8 Microservices 0/4
Modular Monolith vs Microservices

Start with modular monolith! Microservices add network complexity, distributed transactions, eventual consistency. Only decompose when necessary.

Architecture Modularity
Service Communication

Synchronous: OpenFeign (declarative), WebClient (reactive). Asynchronous: Kafka, RabbitMQ for event-driven architecture. RestTemplate is deprecated.

OpenFeign WebClient Kafka
Docker and Containerization

Package Spring Boot JARs in Docker images. Use Buildpacks for Dockerfile-less builds. Docker Compose for multi-container local dev.

Docker Buildpacks Compose
Cloud Deployment Options

PaaS: Render, Railway, Heroku (simple). IaaS: AWS EC2, GCP (control). Free tiers: Oracle Cloud, Render free tier for student projects.

Render Railway AWS

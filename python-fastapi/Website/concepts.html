P0 FastAPI Core 0/5
ASGI and Async Fundamentals

FastAPI is built on Starlette (ASGI framework). ASGI enables async request handling. Use async def for I/O-bound operations, def for CPU-bound tasks.

ASGI Starlette async/await
Path Operations (Decorators)

@app.get(), @app.post(), @app.put(), @app.delete() define API endpoints. Each decorator maps HTTP methods to Python functions. Path determines the URL route.

@app.get @app.post Routes
Dependency Injection System

Depends() injects dependencies into path operations. Enables code reuse, testing, and separation of concerns. Dependencies can have their own dependencies (nested).

Depends DI Injection
Request/Response Models

Pydantic models define request body schema and response structure. FastAPI auto-validates incoming data and serializes responses. Use response_model for output typing.

BaseModel response_model
Automatic OpenAPI Documentation

FastAPI auto-generates OpenAPI schema from type hints. Interactive docs at /docs (Swagger UI) and /redoc. Customize with tags, descriptions, and examples.

/docs /redoc OpenAPI
P1 Data Handling 0/5
Pydantic Models and Validation

Pydantic BaseModel provides automatic validation, serialization, and documentation. Use Field() for constraints like min_length, regex, gt/lt. Validators for custom logic.

BaseModel Field validator
Path Parameters

Extract values from URL path: /users/{user_id}. Type hints enable automatic validation. Use Path() for additional constraints and metadata.

{param} Path() Type Hints
Query Parameters

Parameters after ? in URL: /items?skip=0&limit=10. Optional with default values. Use Query() for validation, alias, and deprecation markers.

Query() Optional Default Values
Request Body Handling

JSON body automatically parsed into Pydantic models. Use Body() for additional metadata. Supports multiple body parameters and nested models.

Body() JSON Nested Models
Response Models and Status Codes

response_model filters output fields. status_code sets HTTP response code. Use status module for readable codes. Multiple response types with responses parameter.

response_model status_code responses
P2 Database Integration 0/5
SQLAlchemy 2.0 with Async

Modern SQLAlchemy with async support. Use create_async_engine and AsyncSession. asyncpg driver for PostgreSQL provides excellent performance.

SQLAlchemy 2.0 AsyncSession asyncpg
ORM Models and Relationships

Define tables with SQLAlchemy ORM. relationship() for associations. mapped_column() for typed columns. ForeignKey for relationships between tables.

DeclarativeBase relationship ForeignKey
Session Management

AsyncSession manages database transactions. Use dependency injection to provide sessions. Context managers ensure proper cleanup. Commit on success, rollback on error.

AsyncSession commit rollback
Alembic Migrations

Database schema versioning and migrations. alembic init, revision --autogenerate, upgrade/downgrade commands. Track changes in version control.

Alembic Migrations autogenerate
Repository Pattern

Abstract database operations behind repository classes. Separates business logic from data access. Easier testing with mock repositories. Single responsibility principle.

Repository Abstraction Clean Code
P3 Security 0/5
OAuth2 Password Flow

Standard authentication flow for first-party apps. User sends username/password, receives access token. OAuth2PasswordBearer handles token extraction from headers.

OAuth2 OAuth2PasswordBearer Token
JWT Tokens

JSON Web Tokens for stateless authentication. Contains header, payload, signature. python-jose library for creation and validation. Set expiration time for security.

JWT python-jose Bearer
Password Hashing

Never store plain text passwords! Use passlib with bcrypt for secure hashing. CryptContext handles hashing and verification. Automatic salt generation.

passlib bcrypt CryptContext
Security Dependencies

Create reusable security dependencies like get_current_user. Chain dependencies for role checks. Raise HTTPException(401/403) for unauthorized access.

get_current_user Depends HTTPException
CORS Configuration

Cross-Origin Resource Sharing for frontend access. CORSMiddleware configures allowed origins, methods, headers. Essential for single-page applications.

CORS CORSMiddleware Origins
P4 Advanced Patterns 0/5
Background Tasks

Run tasks after returning response. BackgroundTasks parameter for email sending, logging, etc. Tasks run in same process - use Celery for heavy work.

BackgroundTasks add_task Async
WebSockets

Real-time bidirectional communication. @app.websocket decorator for endpoints. Handle connect, receive, send events. Great for chat, notifications, live updates.

WebSocket Real-time Bidirectional
Middleware

Process requests before/after route handlers. Add timing, logging, authentication. @app.middleware("http") decorator or BaseHTTPMiddleware class.

Middleware @app.middleware Request/Response
Lifespan Events

Startup and shutdown logic with @asynccontextmanager lifespan. Initialize database pools, load ML models on startup. Clean up resources on shutdown.

lifespan startup shutdown
APIRouter and Sub-applications

Organize routes into modules with APIRouter. Include routers with prefix and tags. Mount sub-applications for microservice patterns or versioning.

APIRouter include_router mount

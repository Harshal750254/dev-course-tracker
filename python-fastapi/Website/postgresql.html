P0 Database Basics 0/5
Relational Database Concepts

Data organized in tables (relations) with rows and columns. Tables have defined schema. Relationships connect tables. ACID properties ensure data integrity.

Tables Rows Columns ACID
Primary Keys and Identifiers

Unique identifier for each row. Can be single column or composite. Common types: SERIAL (auto-increment), UUID, or natural keys. Must be unique and NOT NULL.

PRIMARY KEY SERIAL UUID
Data Types

INTEGER/BIGINT for numbers. VARCHAR(n)/TEXT for strings. BOOLEAN for true/false. TIMESTAMP/DATE for time. NUMERIC for precise decimals. Choose appropriate types for data.

INTEGER VARCHAR TIMESTAMP BOOLEAN
NULL Values and Constraints

NULL means "unknown" - not empty string or zero. NOT NULL constraint requires a value. DEFAULT sets value when not provided. Handle NULLs carefully in queries.

NULL NOT NULL DEFAULT
Installing PostgreSQL

Local install or Docker: docker run -e POSTGRES_PASSWORD=pass -p 5432:5432 postgres. Connect with psql or GUI tools like pgAdmin, DBeaver, or DataGrip.

Docker psql pgAdmin
P1 Essential SQL 0/5
SELECT and WHERE

SELECT columns FROM table WHERE conditions. Use * for all columns (avoid in production). WHERE filters rows with =, <>, <, >, LIKE, IN, BETWEEN, IS NULL.

SELECT WHERE LIKE IN
INSERT, UPDATE, DELETE

INSERT INTO table (cols) VALUES (vals). UPDATE table SET col=val WHERE condition. DELETE FROM table WHERE condition. Always use WHERE with UPDATE/DELETE!

INSERT UPDATE DELETE RETURNING
ORDER BY, LIMIT, OFFSET

ORDER BY col ASC/DESC for sorting. LIMIT n returns first n rows. OFFSET n skips first n rows. Combine for pagination: LIMIT 10 OFFSET 20.

ORDER BY LIMIT OFFSET Pagination
Aggregate Functions

COUNT(*) counts rows. SUM(col), AVG(col) for math. MIN(col), MAX(col) for extremes. NULL values are ignored except COUNT(*). Use with GROUP BY.

COUNT SUM AVG MIN/MAX
GROUP BY and HAVING

GROUP BY groups rows for aggregation. SELECT category, COUNT(*) FROM products GROUP BY category. HAVING filters groups (like WHERE for aggregates).

GROUP BY HAVING Aggregation
P2 Table Design 0/5
CREATE TABLE and ALTER TABLE

CREATE TABLE name (columns with types and constraints). ALTER TABLE ADD/DROP/MODIFY columns. DROP TABLE removes table. Use IF EXISTS/IF NOT EXISTS for safety.

CREATE TABLE ALTER TABLE DROP TABLE
Foreign Keys and Relationships

FOREIGN KEY references another table's primary key. Enforces referential integrity. One-to-Many: FK in "many" table. Many-to-Many: junction table with two FKs.

FOREIGN KEY REFERENCES One-to-Many
Indexes and Performance

CREATE INDEX speeds up WHERE, ORDER BY, JOIN on indexed columns. B-tree default. Trade-off: faster reads, slower writes. EXPLAIN ANALYZE to check query plans.

CREATE INDEX B-tree EXPLAIN ANALYZE
Constraints

UNIQUE ensures no duplicates. CHECK validates values (CHECK (price > 0)). FOREIGN KEY for relationships. Constraints maintain data integrity at database level.

UNIQUE CHECK Integrity
Normalization Basics

Organize data to reduce redundancy. 1NF: no repeating groups. 2NF: no partial dependencies. 3NF: no transitive dependencies. Balance normalization with query performance.

1NF 2NF 3NF Redundancy
P3 Joins and Relationships 0/5
INNER JOIN

Returns rows where both tables have matching values. SELECT * FROM orders JOIN users ON orders.user_id = users.id. Most common join type.

INNER JOIN ON Match
LEFT/RIGHT/FULL JOIN

LEFT JOIN: all left table rows + matching right. RIGHT JOIN: opposite. FULL JOIN: all rows from both. NULLs for non-matching rows. Use for optional relationships.

LEFT JOIN RIGHT JOIN FULL OUTER JOIN
Self-Joins

Join table to itself using aliases. Useful for hierarchical data (employees reporting to managers) or comparing rows within same table.

Self-Join Alias Hierarchy
Junction Tables (Many-to-Many)

Intermediate table for many-to-many relationships. Contains FKs to both tables. Example: user_roles with user_id and role_id. May have additional columns.

Junction Table Many-to-Many Bridge Table
Cascading Actions

ON DELETE CASCADE: delete child rows when parent deleted. ON UPDATE CASCADE: update FKs when PK changes. SET NULL/SET DEFAULT alternatives. Handle carefully!

ON DELETE CASCADE ON UPDATE SET NULL
P4 Advanced Queries 0/5
Subqueries

Query inside another query. In WHERE: WHERE id IN (SELECT ...). In FROM: FROM (SELECT ...) AS subquery. Correlated subqueries reference outer query.

Subquery IN EXISTS
CTEs (WITH Clause)

Common Table Expressions: WITH name AS (query) SELECT ... Named temporary result sets. More readable than nested subqueries. Can be recursive for hierarchies.

WITH CTE RECURSIVE
UNION, INTERSECT, EXCEPT

UNION: combine results (removes duplicates, UNION ALL keeps them). INTERSECT: rows in both queries. EXCEPT: rows in first but not second. Columns must match.

UNION INTERSECT EXCEPT
Window Functions

Calculations across row sets without grouping. ROW_NUMBER(), RANK(), DENSE_RANK() for ordering. LAG/LEAD access adjacent rows. OVER(PARTITION BY ... ORDER BY ...).

ROW_NUMBER RANK LAG/LEAD OVER
CASE Expressions

Conditional logic in SQL. CASE WHEN condition THEN result ELSE default END. Use for conditional columns, bucketing data, or dynamic ordering.

CASE WHEN THEN Conditional
P5 PostgreSQL-Specific Features 0/5
JSON/JSONB Columns

Store JSON data in columns. JSONB is binary, faster for queries. Access with -> (JSON) and ->> (text). @> for containment. GIN indexes for fast searches.

JSONB -> ->> @>
Arrays

Array column type: INTEGER[], TEXT[]. Access with [index] (1-based). ANY/ALL for comparisons. array_agg() to build arrays. Useful for tags, preferences.

ARRAY[] ANY array_agg
Full-Text Search

tsvector for searchable documents. tsquery for search terms. @@ operator for matching. GIN/GiST indexes. to_tsvector(), plainto_tsquery() functions.

tsvector tsquery @@ GIN
Primary Key Strategies

SERIAL: auto-increment integer. BIGSERIAL for large tables. UUID: globally unique, good for distributed systems. IDENTITY (SQL standard alternative to SERIAL).

SERIAL UUID IDENTITY
ENUM Types

CREATE TYPE status AS ENUM ('pending', 'active', 'done'). Type-safe, ordered values. Efficient storage. Harder to modify than check constraints. Use for stable value sets.

CREATE TYPE ENUM Type Safety
P6 Administration Basics 0/5
Creating Databases and Users

CREATE DATABASE mydb. CREATE USER username WITH PASSWORD 'pass'. CREATE ROLE for groups. Each app should have its own user with minimal permissions.

CREATE DATABASE CREATE USER CREATE ROLE
Permissions (GRANT/REVOKE)

GRANT SELECT, INSERT ON table TO user. REVOKE DELETE ON table FROM user. Permissions: SELECT, INSERT, UPDATE, DELETE, ALL. Grant on schemas for broader access.

GRANT REVOKE Permissions
Backup and Restore

pg_dump dbname > backup.sql for logical backup. pg_restore for restoring. pg_dumpall for entire cluster. Schedule regular backups. Test restores!

pg_dump pg_restore Backup
Connection Strings

postgresql://user:password@host:port/database. Use environment variables for credentials. Connection pooling with pgBouncer for production. SSL for security.

Connection String pgBouncer SSL
psql Command-Line Basics

psql -U user -d database to connect. \l list databases. \dt list tables. \d table describe table. \q quit. \i file.sql execute SQL file. Essential for scripting.

psql \dt \d \l

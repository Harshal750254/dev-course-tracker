Critical OOP Fundamentals 0/4
Classes, Inheritance, Composition

Classes define object blueprints. Inheritance (class Child(Parent)) for IS-A relationships. Composition (HAS-A) is often preferred for flexibility and loose coupling.

class super() __init__
Dataclasses (@dataclass)

Decorator that auto-generates __init__, __repr__, __eq__. Reduces boilerplate for data-holding classes. Use frozen=True for immutability. Perfect for DTOs and configs.

@dataclass field() frozen
Abstract Base Classes (ABC)

Define interfaces with abc module. @abstractmethod enforces implementation in subclasses. Use for dependency inversion and testable architecture.

ABC @abstractmethod Interface
Magic/Dunder Methods

Special methods like __str__, __repr__, __eq__, __hash__. Enable operator overloading and built-in function integration. __enter__/__exit__ for context managers.

__str__ __repr__ __eq__
Critical Modern Python 0/4
Type Hints and typing Module

Static type annotations for better IDE support and documentation. List[str], Dict[str, int], Optional[T], Union[A, B]. Essential for FastAPI and Pydantic.

typing Optional Union
F-strings and String Formatting

f"Hello {name}" for readable string interpolation. Support expressions: f"{price:.2f}". Debug with f"{var=}". Preferred over .format() and % formatting.

f-string Format Spec =
Context Managers (with)

Automatic resource management with with statement. Ensures cleanup even on exceptions. Use for files, database connections, locks. Create with @contextmanager or __enter__/__exit__.

with @contextmanager Resource
Walrus Operator (:=)

Assignment expression: if (n := len(data)) > 10. Assigns and returns value in one expression. Useful in while loops and comprehensions. Python 3.8+.

:= Python 3.8+ Assignment
Backend Essential Data Structures 0/4
Lists, Dicts, Sets, Tuples

List: ordered, mutable [1, 2, 3]. Dict: key-value O(1) lookup {"a": 1}. Set: unique elements {1, 2}. Tuple: immutable (1, 2). Choose based on mutability and lookup needs.

list dict set tuple
List/Dict Comprehensions

Concise syntax: [x*2 for x in range(10) if x > 5]. Dict: {k: v for k, v in items}. Set: {x for x in data}. More readable than map/filter for simple cases.

[expr for x in iter] {k: v} Filtering
Collections Module

defaultdict: dict with default factory. Counter: count hashable objects. deque: efficient append/pop from both ends. OrderedDict: maintains insertion order (less needed in 3.7+).

defaultdict Counter deque
Named Tuples

Tuples with named fields. from collections import namedtuple or typing.NamedTuple. Lightweight alternative to classes. Immutable and memory efficient.

namedtuple NamedTuple Immutable
Backend Essential Async Programming 0/4
async/await Syntax

async def defines coroutine. await suspends until result ready. Only await inside async functions. Enables concurrent I/O without threads.

async def await Coroutine
asyncio Basics

asyncio.run() starts event loop. asyncio.gather() runs multiple coroutines concurrently. asyncio.create_task() schedules coroutine. asyncio.sleep() for non-blocking delays.

asyncio.run gather create_task
Coroutines and Tasks

Coroutine: async function that can be paused. Task: scheduled coroutine running in event loop. await coroutine vs create_task() for concurrent execution.

Coroutine Task Concurrent
Event Loops

Core of asyncio - schedules and runs coroutines. Single-threaded concurrency via cooperative multitasking. asyncio.get_event_loop() or asyncio.run() manages it.

Event Loop Single Thread Scheduling
Important Functional Programming 0/4
Lambda Expressions

Anonymous functions: lambda x: x * 2. Single expression only. Use for simple callbacks, sorting keys, map/filter. Prefer named functions for complex logic.

lambda Anonymous key=
map, filter, reduce

map(func, iterable) applies function to each element. filter(func, iterable) keeps truthy results. reduce() from functools for accumulation. Often replaced by comprehensions.

map filter functools.reduce
Generators and Iterators

yield creates generator - lazy evaluation, memory efficient. Generator expressions: (x for x in range(10)). __iter__ and __next__ for custom iterators.

yield Generator Lazy
Decorators

Functions that modify other functions. @decorator syntax wraps target. Use functools.wraps to preserve metadata. Common: @staticmethod, @classmethod, @property.

@decorator functools.wraps Wrapper
Good to Know Exception Handling 0/4
try/except/finally

try for risky code, except catches errors, finally always runs. Catch specific exceptions first. Use else for code that runs on success only.

try except finally
Custom Exceptions

Inherit from Exception or specific built-in exceptions. Add custom attributes for context. class NotFoundError(Exception): pass. Use for domain-specific errors.

class Error(Exception) raise Domain
Exception Chaining

raise NewError() from original_error preserves traceback. Shows both exceptions in output. raise from None to suppress context. Python 3 feature.

raise from __cause__ Traceback
Built-in Exceptions

ValueError: wrong value. TypeError: wrong type. KeyError: missing dict key. AttributeError: missing attribute. FileNotFoundError: file doesn't exist.

ValueError TypeError KeyError
Good to Know Advanced Concepts 0/4
Metaclasses

Classes that create classes. type is the default metaclass. Use class Meta(type) for custom behavior. Rarely needed - consider decorators or __init_subclass__ first.

type __new__ metaclass=
Descriptors

Objects defining __get__, __set__, __delete__. Power behind @property, @classmethod. Enable attribute access customization. Used in ORMs and validators.

__get__ __set__ Property
__slots__

Restrict instance attributes to declared names. Saves memory by avoiding __dict__. __slots__ = ['x', 'y']. Trade flexibility for efficiency in many instances.

__slots__ Memory Performance
GIL and Multiprocessing

Global Interpreter Lock prevents true parallelism in threads. Use multiprocessing for CPU-bound tasks. Threading fine for I/O-bound. concurrent.futures for high-level API.

GIL multiprocessing ProcessPoolExecutor
